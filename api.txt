    SELECT 
    ur.userId,
    qs.skillId,
    COUNT(*) AS attempts,
    SUM(CASE WHEN ur.isCorrect = 1 THEN 1 ELSE 0 END) AS correct,
    CAST(SUM(CASE WHEN ur.isCorrect = 1 THEN 1 ELSE 0 END) AS FLOAT) / COUNT(*) AS accuracy,
    CASE 
        WHEN CAST(SUM(CASE WHEN ur.isCorrect = 1 THEN 1 ELSE 0 END) AS FLOAT) / COUNT(*) < 0.6 
        THEN 1 ELSE 0 
    END AS isWeak
FROM UserResults ur
JOIN QuestionSkills qs ON ur.questionId = qs.questionId
GROUP BY ur.userId, qs.skillId;

+++++++++++++++++++++++++++++++++


1️⃣ Lấy kết quả làm bài của user

Bảng UserResults: chứa userId, questionId, isCorrect.

Đây là data gốc để biết user làm đúng/sai câu nào.

👉 Từ đây bạn join qua Questions và QuestionSkills để biết mỗi câu thuộc skill nào.

2️⃣ Tính toán theo skill

Gom group theo (userId, skillId) từ join ở trên.

Tính:

attempts = COUNT(questionId)

correct = SUM(isCorrect)

accuracy = correct / attempts

👉 Nếu accuracy < 0.6 → gắn nhãn Weak, ngược lại Strong.
(Cái này chính là input để train/test với Naive Bayes trong predict_hybrid.py.)

3️⃣ Xác định skill yếu của user

Dùng output từ predict_hybrid.py để lấy danh sách skill Weak.

Ví dụ: User 6 → Weak: Vocabulary.

4️⃣ Lấy các câu sai trong skill yếu

Từ UserResults lọc isCorrect = 0

Join với QuestionSkills để chỉ lấy những câu sai thuộc skill Weak.

Đây sẽ là “anchor question(s)” để tìm tương tự.

5️⃣ Gợi ý câu hỏi mới

Lấy embedding của các “anchor questions”.

So sánh với vector trong bảng QuestionEmbeddings (nếu bạn có bảng đó, hoặc có thể bổ sung dựa trên Questions).

Chọn top-k câu hỏi tương tự, cùng skill, nhưng loại trừ những câu user đã làm (dựa vào UserResults).

6️⃣ Trả kết quả

Output cuối cùng là danh sách câu hỏi mới:

questionId

question

skill

similarity score

👉 Gửi danh sách này cho frontend để hiển thị câu hỏi luyện tập tiếp theo.

✅ Vậy thứ tự làm việc trong DB là:
UserResults → Questions → QuestionSkills → Skills → (QuestionEmbeddings)

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

✅ Hôm nay bạn đã làm xong

Phát hiện kỹ năng yếu (Weak Skills)

Viết predict_hybrid.py:

Train global model (Naive Bayes) từ tất cả user.

Train personal model cho từng user nếu có đủ dữ liệu.

Hybrid rule:

< 10 attempts → dùng global model.

>= 10 attempts → dùng personal model.

Output: { skillName: Weak/Strong (global|personal) }.

Lấy anchor câu sai trong skill yếu

Query UserResults + Questions + QuestionSkills → tìm câu user làm sai trong skill yếu.

Chọn 1 câu làm “anchor” để tìm câu tương tự.

Gợi ý câu hỏi mới bằng kNN

Viết findSimilar.js:

Sinh embedding cho câu anchor.

So sánh với vector trong QuestionEmbeddings.

Dùng cosine similarity để sort.

Trả về top-k câu hỏi → chính là kNN.

Fix để output ra JSON.

Tích hợp gọi từ Python → Node qua subprocess.

Test end-to-end

Chạy python ml/predict_hybrid.py.

Kết quả: dự đoán skill yếu, chọn anchor, gọi findSimilar.js, trả về câu hỏi gợi ý dạng JSON.

🚀 Ngày mai dự tính làm tiếp

Cải thiện chất lượng gợi ý

Lọc duplicate kết quả trong findSimilar.js.

Loại bỏ các câu user đã làm (join với UserResults).

Ưu tiên câu hỏi theo độ khó phù hợp (easy/medium/hard nếu DB có).

Chuẩn hoá dữ liệu output

Parse JSON trong Python thay vì in raw string.

Chuẩn bị format để dễ đưa vào API/Frontend.

Tích hợp API

Tạo API (Flask/FastAPI hoặc trong NodeJS Express) endpoint /recommend/:userId.

Input: userId.

Output: Weak skills + danh sách câu hỏi gợi ý.

Optional nâng cao

Gợi ý nhiều anchor hơn (không chỉ 1 câu sai, mà lấy top-N câu sai gần đây).

Kết hợp embedding với metadata (ví dụ: chỉ tìm trong đúng skill yếu, đúng Part).

👉 Như vậy, hôm nay bạn đã hoàn thành pipeline core: detect Weak Skills (Naive Bayes) + Recommend Questions (kNN).
Ngày mai chủ yếu là làm sạch, tối ưu và mở API để frontend xài được.
